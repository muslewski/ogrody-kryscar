# Specs

Design specs produced by the brainstorming flow. Each file is named `YYYY-MM-DD-<feature>-design.md`.

**Convention:** specs are written during brainstorming, committed to `main` immediately (not the feature branch), and consumed (not edited) once a plan is written from them. To supersede a spec, create a new one and set `status: superseded` in the old one's frontmatter — never edit a consumed spec.
