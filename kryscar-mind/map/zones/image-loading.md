## Purpose
Instant blurred-preview image loading for self-hosted garden images: a committed, generated `blurDataURL` map keyed by public path + a server-side `BlurImage` wrapper over `next/image` (`placeholder="blur"`). The map now also holds the 12 `/img/projects/*` before/after entries consumed by the [[realizacje]] gallery slider.
## Anchors
`BlurImage`, `BLUR_DATA`, `hasBlurImage`, `scripts/gen-blur.mjs`.
## Invariants
`blur-data.ts` is generated (DO NOT EDIT); graceful `placeholder="empty"` fallback for any image without an entry. `BlurImage` is a server component so the map stays server-side.
## hasBlurImage guard
`hasBlurImage(src)` is an exported type-guard (`src is string`) that returns true only when `src` is in `BLUR_DATA` — i.e. the file exists and the blur map has been regenerated. Used by [[winter-services]] (`/zima/[usluga]` hero and `WinterServiceCard`) to gate optional images so not-yet-fetched photos fall back gracefully. Rationale → [[winter-image-blur-gating]].
